# 需求拆解：模块 → 子功能 → 负责组 → 关键端点/表

> 源自《"趣聚"平台需求 - v2》。每条是一个可领取的 feature 单元。
> 领取后照 `docs/workflow.md` 写 spec→plan→tasks→implement。负责组见 `work-assignment.md`。
> 编号用于在 spec / PR 中引用（如 `R2.3 活动发现`）。

## R1. 用户信息管理（负责：用户组）

| 编号 | 子功能 | 关键点 | 端点(示例) | 表 |
|---|---|---|---|---|
| R1.1 | 注册 | 个人/商家；邮箱+密码；邮件激活链接；商家传营业执照 | `POST /v1/auth/register` `GET /v1/auth/activate` | user, email_activation, merchant_profile |
| R1.2 | 登录 | JWT | `POST /v1/auth/login` | user |
| R1.3 | 个人资料 | 头像/昵称(全局唯一)/性别/生日/签名/兴趣标签 | `GET/PUT /v1/users/me` | user, user_interest_tag |
| R1.4 | 商家资料 | 名称/昵称/关注领域 | `PUT /v1/merchants/me` | merchant_profile |

## R2. 活动全生命周期（负责：活动组，建议 4 人分两摊）

### R2-A 创建 / 审核 / AI（活动组 a）
| 编号 | 子功能 | 关键点 | 表 |
|---|---|---|---|
| R2.1 | 用户创建活动 | 名称/标签/简介/时间/**地图选点**/人数上限/报名截止 | activity |
| R2.2 | AI 活动策划 | 输入主题/类型 → AI 生成基础信息，可逐项改 | activity（+AI 网关） |
| R2.3 | 模板与克隆 | 6 类模板；历史活动克隆改差异字段 | activity_template, activity |
| R2.4 | 草稿保存与提交 | 随时存草稿；提交进审核流 | activity(draft) |
| R2.5 | AI 内容安全审核 | 违法/低俗/危险；通过直接发，存疑转人工 | activity_audit_log（+AI 网关） |
| R2.6 | 人工审核 | 报名>50 人 或 AI 存疑 → 人工；通过/驳回/改 | activity_audit_log |

## R5. 后台管理（负责：后台组）

| 编号 | 子功能 | 关键点 | 表 |
|---|---|---|---|
| R5.1 | 管理员登录 | 预置账号，无注册，可改本人密码 | admin |
| R5.2 | 用户管理 | 查询个人/商家；看信息+状态+其活动/小队；不可改资料/密码 | user |

| R5.6 | 小队管理 | 查询；看成员/活动/举报；停用(填原因)；可恢复 | team, moderation_action, report |

## 横切（负责：平台核心组，Day0–2 先行）

- 统一信封 / 全局异常 / 分页 / 鉴权(JWT) / 基类
- **AI 网关**：把 R2.2 / R2.5 / R2.17 的 LLM/视觉调用收口到一个可 mock 的模块
- 第三方适配：**地图**(高德，选点/附近/距离/地图模式)、**邮件**(激活)、**文件存储**(头像/活动图/群文件/相册)、**短信**
- **即时通讯**走 WebSocket（R3.4 + R4.4 群聊）
- CI / 契约校验 / 脚手架

## 非功能（全员遵守）
- 启动至可操作 < 5s；后台业务接口 < 2000ms（第三方除外）。
