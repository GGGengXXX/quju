# 趣聚 QuJu 项目宪法（Constitution）

> 本项目不可妥协的底线原则。所有 spec / plan / 实现都必须符合本宪法。
> 改动本宪法需平台核心组共识。对应 GitHub Spec Kit 的 `constitution`。

## 一、契约优先（不可妥协）

1. `contracts/`（OpenAPI + DB schema + 枚举 + 错误码）是前后端唯一事实来源。
2. 任何契约变更：先写 ADR → 平台核心组评审 → 升版本 → 通知全员重新生成。
3. 实现与契约不一致，一律以契约为准；不一致即缺陷。

## 二、隔离与边界

4. 一个 feature = 一个分支 = 一个 worktree；只改自己负责的目录/表/包。
5. 每个模块单元职责单一、接口清晰、可独立测试。文件过大是"做太多"的信号。

## 三、可验证才算完成

6. 每次实现都附带可运行的验证（测试/构建/截图），并展示证据而非断言成功。
7. Definition of Done（根 AGENTS §6）未全部满足，PR 不可合并。
8. 合并前过一次"对抗式自审"（新上下文 review diff 对照规格）。

## 四、安全与质量底线

9. 密钥只走环境变量，绝不入库。
10. 输入校验、鉴权、越权防护是默认项，不是可选项。
11. 内容安全：用户生成内容（活动、动态、图片）须经审核流程（AI + 必要时人工）。

## 五、性能预算（硬指标）

12. App/Web 启动至可操作 < 5s；后台业务接口单次 < 2000ms（第三方除外）。
13. 列表必分页；地理查询走空间索引；热点走缓存。

## 六、协作

14. 规则写在 `AGENTS.md`（工具中立），Claude Code 经 `CLAUDE.md` 导入；所有工具读同一套规则。
15. 沟通靠仓库内文档（spec / ADR / PR），不靠口头与记忆。
