# 实现计划 Plan：R4 小队模块 dev 缺陷修复

## 1. 方案概述
沿用现有团队模块的手写 API + 单页工作台模式，本次尽量把修复收敛在前端：通过列表按钮守卫和详情加载短路解决未加入误报错，通过 URL 规范化解决本地开发地址渲染问题，通过登录后轮询已加入小队公告并结合 `@所有人` / `@昵称` 规则实现提醒。

## 2. 后端改动
- 无后端契约或表结构改动。
- 继续复用现有 `GET /v1/teams` 与 `GET /v1/teams/{id}/announcements` 数据。

## 3. 前端改动
- `frontend/src/views/team/TeamHub.vue`
  - 删除未加入小队的“查看”按钮。
  - 给详情刷新加成员态守卫。
  - 相册/动态提交前做 URL 规范化，展示时统一用规范化地址。
  - 公告输入提示 `@所有人` / `@昵称` 语法。
- `frontend/src/App.vue`
  - 登录后确保加载当前用户信息。
  - 轮询已加入小队公告，按本地已读游标判断是否有新的相关公告。
  - 用 Element Plus 右上角通知弹窗提示。

## 4. 契约依赖
- 只使用已有团队接口，不修改 `contracts/openapi.yaml`。
- `@` 语义在前端解析，不新增表字段。

## 5. 测试策略
- 前端：`npm run type-check`、`npm run build`
- 手工验证：未加入列表态、加入后详情、相册/动态 URL 渲染、公告 @ 提醒。

## 6. 风险与依赖
- `file://` 本地文件路径是否可显示仍受浏览器安全策略限制，本次会做地址规范化和更合理的渲染，但不实现真正的本地文件上传。
- 公告提醒采用轮询，不是实时推送；符合本次 bugfix 范围，不扩展到 WebSocket。
