# 规格 Spec：R4 小队模块 dev 缺陷修复

- **模块**：team
- **负责人**：付东淏
- **关联需求**：`docs/requirements-breakdown.md` 中 `R4.2`、`R4.4`、`R4.6`
- **分支**：`dev`

## 1. 目标（一句话）
这个修复让小队页面的查看权限、图片 URL 渲染和公告 @ 提醒行为与 R4 需求一致，避免成员态误报错并补齐关键提醒体验。

## 2. 验收标准（可测试，逐条）
- [ ] 给定用户未加入某个小队，当进入小队列表时，则该卡片不展示“查看”按钮，且不会因为误调详情受保护接口而出现 `no_team_permission`。
- [ ] 给定成员在相册或动态中提交可访问的本地开发 URL（如 `localhost:3000/a.png`、`127.0.0.1:8080/b.jpg`、`file:///...`），当保存并展示时，则前端会做 URL 规范化并按规范地址渲染。
- [ ] 给定队长或管理员发布带 `@所有人` 或 `@成员昵称` 的公告，当其他相关成员登录进入页面后，则右上角会弹出“XX 小队有新的和你相关的公告”提示。
- [ ] 给定发布者本人登录，则不会给自己弹出自己发出的相关公告提醒。
- [ ] 非功能：本次不修改 `contracts/`、不新增数据表、不扩展其他模块逻辑。

## 3. 涉及的契约（必须先确认/补充 contracts/）
- **端点**：`GET /v1/teams`、`GET /v1/teams/{id}/announcements`、`POST /v1/teams/{id}/announcements`、`POST /v1/teams/{id}/album`、`POST /v1/teams/{id}/moments`
- **DB 表/列**：只读/沿用 `team`、`team_member`、`team_announcement`、`team_album_photo`、`team_moment`
- **枚举 / 错误码**：沿用 `TeamRole`、`5004 no_team_permission`
- 若需改契约：本次不改 `contracts/`

## 4. 范围
- **做**：修正小队页按钮与详情加载守卫；补前端 URL 规范化；补登录后小队公告 @ 提醒。
- **不做（Out of scope）**：新增通知表、消息中心、WebSocket 推送、OSS 文件上传、改动其他模块。

## 5. UI / 交互（前端）
- 页面 / 组件：`frontend/src/views/team/TeamHub.vue`、`frontend/src/App.vue`
- 关键交互、状态、空/错/加载态：未加入时仅显示“加入”；公告输入支持 `@所有人` / `@昵称` 约定；提醒使用右上角通知弹窗。

## 6. 端到端验证步骤
1. 登录 A，创建或找到一个自己未加入的小队，确认只显示“加入”不显示“查看”。
2. 加入小队后再打开详情，确认不再出现权限报错。
3. 在相册和动态里输入 `localhost:3000/demo.png` 或 `file:///Users/.../demo.png`，确认保存后展示使用规范化地址。
4. 队长发布 `@所有人` 或 `@某成员昵称` 公告。
5. 被 @ 的成员重新登录进入页面，确认右上角弹出对应小队公告提醒；发布者本人无提醒。
